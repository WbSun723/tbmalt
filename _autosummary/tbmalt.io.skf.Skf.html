2.1.3.2.1. Skf

class Skf(atom_pair, hamiltonian, overlap, grid, r_spline=None, r_poly=None, hubbard_us=None, on_sites=None, occupations=None, mass=None)[source]

Bases: object

Slater-Koster file parser.

This class handles the parsing of DFTB+ skf formatted Slater-Koster files, and their binary analogs. Data can be read from and saved to files using the read & write methods. Reading a file will return an Skf instance holding all stored data.

Parameters:

  • atom_pair (Tensor) – Atomic numbers of the elements associated with the interaction.

  • hamiltonian (Dict[Tuple[int, int], Tensor]) – Dictionary keyed by azimuthal-number-pairs (ℓ₁, ℓ₂) and valued by m×d Hamiltonian integral tensors; where m and d iterate over bond-order (σ, π, etc.) and distances respectively.

  • overlap (Dict[Tuple[int, int], Tensor]) – Dictionary storing the overlap integrals in a manner akin to hamiltonian.

  • grid (Tensor) – Distances at which the hamiltonian & overlap elements were evaluated.

  • r_spline (Optional[RSpline]) – A RSpline object detailing the repulsive spline. [DEFAULT=None]

  • r_poly (Optional[RPoly]) – A RPoly object detailing the repulsive polynomial. [DEFAULT=None]

  • on_sites (Optional[Tensor]) – On site terms, homo-atomic systems only. [DEFAULT=None]

  • hubbard_us (Optional[Tensor]) – Hubbard U terms, homo-atomic systems only. [DEFAULT=None]

  • mass (Optional[Tensor]) – Atomic mass, homo-atomic systems only. [DEFAULT=None]

  • occupations (Optional[Tensor]) – Occupations of the orbitals, homo-atomic systems only. [DEFAULT=None]

Examples

Examples of reading and writing.

>>> import urllib.request, tarfile
>>> from os.path import join
>>> from tbmalt.io.skf import Skf
>>> link = 'https://github.com/dftbparams/auorg/releases/download/v1.1.0/auorg-1-1.tar.xz'
>>> urllib.request.urlretrieve(link, 'auorg-1-1.tar.xz')
>>> tartmp = tarfile.open('auorg-1-1.tar.xz')
>>> tartmp.extractall()
>>> cc = Skf.from_skf(join('auorg-1-1', 'C-C.skf'))
>>> print(cc.hamiltonian.keys())
dict_keys([(0, 0), (0, 1), (1, 1)])

use auorg-1-1 to generate binary h5 and read the binary file

>>> cc.write('tmp.h5')
>>> cch5 = Skf.read('tmp.h5')
>>> print(cch5.hamiltonian.keys())
dict_keys([(0, 0), (0, 1), (1, 1)])

Note Slater-Koster parameter sets can be downloaded and parsed into an HDF5 file using the method tbmalt.tools.downloaders.download_dftb_parameter_set.

atomic

True if the system contains atomic data, only relevant to the homo-atomic cases.

Notes

HOMO atomic systems commonly, but not always, include additional “atomic” data; namely atomic mass, on-site terms, occupations, and the Hubbard-U values. These can be optionally specified using the mass, on_sites, occupations, and hubbard_us attributes respectively. However, these attributes are mutually inclusive, i.e. either all are specified or none are. Furthermore, values contained within such tensors should be ordered from the lowest azimuthal number to highest, where applicable.

Further information regarding the skf file format specification can be found in the document: “Format of the v1.0 Slater-Koster Files”.

Warning

This may fail to parse files which do not strictly adhere to the skf file format. Some skf files, such as those from the “pbc” parameter set, contain non-trivial errors in them, e.g. incorrectly specified number of grid points. Such files require fixing before they can be read in.

The atom_pair argument is order sensitive, i.e. [6, 7] ≠ [7, 6]. For example, the p-orbital of the s-p-σ interaction would be located on N when atom_pair is [6, 7] but on C when it is [7, 6].

Raises:

ValueError – if some but not all atomic attributes are specified. See the Notes section for more details.

classmethod from_skf(path, dtype=None, device=None)[source]

Parse and skf file into an Skf instance.

File names should follow the naming convention X-Y.skf where X & Y are the chemical symbols of the associated elements. However, any file which ends in X.Y will be successfully parsed (where “.” is any character (including no character)).

Parameters:

  • path (str) – Path to the target skf file.

  • device (Optional[device]) – Device on which to place tensors. [DEFAULT=None]

  • dtype (Optional[dtype]) – dtype to be used for floating point tensors. [DEFAULT=None]
